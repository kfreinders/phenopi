\section{Introduction}

\subsection{Purpose}

Phenopi is an image-based plant phenotyping system for controlled-environment
experiments. It is designed for repeated RGB imaging of plants and for
extracting quantitative traits such as canopy area and growth over time. The
system combines automated image acquisition, image analysis, quality control,
and growth analysis in a modular workflow. It was developed as a low-cost
alternative to commercial phenotyping platforms and can run on inexpensive
hardware such as a Raspberry Pi camera setup. Users do not need extensive
programming experience, but Phenopi is currently command-line based and assumes
basic familiarity with running terminal commands and working with text-based
configuration files.

\subsection{What Phenopi Does}

Phenopi supports a complete basic workflow for image-based plant phenotyping.
During an experiment, a Raspberry Pi captures images at scheduled time points
and stores them using timestamped filenames. After acquisition, the analysis
pipeline processes the images, segments plant canopies, extracts traits, and
writes the results to output tables. The system also produces diagnostic
outputs that help users check image quality and segmentation accuracy before
interpreting the biological results. Growth analysis tools can then be used to
summarize canopy development over time.

\subsection{What This Manual Covers}

This manual explains how to install Phenopi for automated image acquisition,
prepare an experiment, run the scheduler, process captured images, inspect
quality-control outputs, and perform basic growth analysis. The manual focuses
on practical use. It describes the files users need to edit, the commands they
need to run, the outputs they should expect, and the checks they should perform
when something goes wrong. Detailed implementation notes are kept to a minimum.
Where internal behaviour is described, it is included only when it helps users
understand how to operate or troubleshoot the system.

\section{System Overview}

\subsection{Overview}

Phenopi is organized in three main parts: image acquisition, image analysis,
and biological analysis. Together, these parts form a workflow that starts with
an experiment configuration file and ends with trait tables, diagnostic
outputs, and growth analysis. The sections below give a brief overview of each
part of the system. Detailed instructions are provided in later chapters.

\subsection{Image Acquisition}

The image acquisition subsystem captures images at predefined times during an
experiment. It is controlled by a lightweight scheduler running on a Raspberry
Pi, and installed as a service (meaning it's always running in the background).
The scheduler reads a user-defined schedule and runs capture jobs within their
permitted time windows. It also keeps track of completed captures, which helps
prevent duplicate images when the system is restarted or interrupted. Captured
images are timestamped automatically and stored using a standard filename
convention.

\subsection{Image Analysis}

After acquisition, RGB images are processed by the canopy analysis pipeline.
This pipeline prepares the images, assigns regions of interest, segments the
plants, extracts basic morphological traits, and writes the results to output
tables. The pipeline is intended for batch processing of complete experiments.
Where possible, images are processed in parallel to reduce analysis time.

\subsection{Quality Control}

Quality control is an important part of the Phenopi workflow. Automated image
analysis can fail when lighting changes, plants move, segmentation settings are
unsuitable, or images are missing. Phenopi produces diagnostic outputs that
help users inspect these problems. These include segmentation overlays and
colour diagnostics. Users should review these outputs before interpreting the
biological results.

\subsection{Biological Analysis}

The extracted trait tables are used for downstream biological analysis. Current
analysis modules support growth curves, individual plant trajectories and
growth rate estimation. These analyses are intended as a starting point for
interpreting experiment results. More specialized statistical analyses may
still need to be performed separately, depending on the experimental design.

\section{Installation}

\subsection{System Requirements}

The image acquisition subsystem requires a Raspberry Pi 4 or newer, a Raspberry
Pi Camera Module v3, Raspberry Pi OS Bookworm or newer, and Picamera2. The
analysis software has been developed under Linux. It may also work on other
platforms supported by the required Python libraries, but Linux is the
recommended environment.

\subsection{Installation}

\subsection{Installation for Automated Acquisition}

For automated image acquisition, Phenopi should be installed as a systemd
service on the Raspberry Pi. This allows the scheduler to start automatically
after boot and continue running in the background.

The provided service file is located at:

\begin{verbatim}
deploy/phenopi-scheduler.service
\end{verbatim}

The project repository should be located at \texttt{/home/phenopi/phenotyping},
and the Python virtual environment should be located at
\texttt{/home/phenopi/venvs/phenopi}.

Create a dedicated user for running the scheduler.

\begin{verbatim}
sudo useradd -m -s /bin/bash phenopi
\end{verbatim}

Create the virtual environment directory.

\begin{verbatim}
sudo -u phenopi mkdir -p /home/phenopi/venvs
\end{verbatim}

Clone the Phenopi repository into the expected location.

\begin{verbatim}
sudo -u phenopi git clone https://github.com/kfreinders/phenopi.git 
/home/phenopi/phenotyping
\end{verbatim}

Install Picamera2 using the Raspberry Pi package manager.

\begin{verbatim}
sudo apt update
sudo apt install -y python3-picamera2
\end{verbatim}

Create a Python virtual environment. On Raspberry Pi OS, using
\texttt{--system-site-packages} makes packages installed through \texttt{apt},
such as Picamera2, available inside the environment.

\begin{verbatim}
sudo -u phenopi python3 -m venv --system-site-packages 
/home/phenopi/venvs/phenopi
\end{verbatim}

Install the required Python packages.

\begin{verbatim}
sudo -u phenopi /home/phenopi/venvs/phenopi/bin/pip install -r 
/home/phenopi/phenotyping/requirements.txt
\end{verbatim}

Before installing the service, check that the scheduler command runs correctly.

\begin{verbatim}
cd /home/phenopi/phenotyping
sudo -u phenopi /home/phenopi/venvs/phenopi/bin/python 
-m scripts.scheduling.run_scheduler --help
\end{verbatim}

If the help page is displayed, copy the service file to the systemd service
directory.

\begin{verbatim}
sudo cp /home/phenopi/phenotyping/deploy/phenopi-scheduler.service 
/etc/systemd/system/phenopi-scheduler.service
\end{verbatim}

Reload systemd, enable the service, and start it.

\begin{verbatim}
sudo systemctl daemon-reload
sudo systemctl enable phenopi-scheduler.service
sudo systemctl start phenopi-scheduler.service
\end{verbatim}

Check whether the service is running.

\begin{verbatim}
sudo systemctl status phenopi-scheduler.service
\end{verbatim}

The service logs can be viewed using \texttt{journalctl}.

\begin{verbatim}
sudo journalctl -u phenopi-scheduler.service -f
\end{verbatim}

To stop the scheduler manually, run:

\begin{verbatim}
sudo systemctl stop phenopi-scheduler.service
\end{verbatim}

To restart the scheduler after changing the configuration or updating the code,
run:

\begin{verbatim}
sudo systemctl restart phenopi-scheduler.service
\end{verbatim}

If the service fails to start, first check that the paths in
\texttt{deploy/phenopi-scheduler.service} match the actual installation paths.
In particular, verify \texttt{WorkingDirectory}, \texttt{ExecStart}, and the
permissions of the configured output directories.
